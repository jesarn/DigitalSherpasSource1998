Parameters

The param tags are the lines of HTML which customize the applet to your needs. Following is a list of all the parameter options which are currently applicable. Be sure to use double-quotes around the parameter's value if the value contains any whitespace. Most parameters have defaults in case you choose not to specify them... but some parameters must be included.

  • COPYRIGHT: This parameter MUST be included and MUST have a value of: 
    "Wordsearch applet, Copyright 1997 Sun Microsystems, Inc., all rights reserved."

  • DOCUMENTATION: This parameter MUST be included and MUST have a value of: 
    "Documentation found at: http://www.xm.com/cafe/"

  • BACKGROUND: This parameter points to an image file which will appear on the puzzle board behind the letters. Be sure to guidelines about images on the FAQ page.

  • BGCOLOR: This parameter indicates the background color of the applet's list and buttons (if you want the background color of the board area to be different, use the PUZZLE.BGCOLOR parameter). See guidelines for colors to learn acceptable values for this parameter.

  • CELLSIZE: This parameter indicates the pixel cellsize allocated for each letter in the board of the puzzle. The default is 24.

  • FGCOLOR: This parameter indicates the foreground color of the applet's list and buttons (if you want the foreground color of the board area to be different, use the PUZZLE.FGCOLOR parameter). See guidelines for colors to learn acceptable values for this parameter.

  • FONTNAME: This parameter indicates font face used in the puzzle. Be sure to read the FAQ section on fonts.

  • FONTSIZE: This parameter indicates font size used in the puzzle. Be sure to read the FAQ section on fonts.

  • FONTSTYLE: This parameter indicates font style used in the puzzle. Be sure to read the FAQ section on fonts.

  • FOUND.SOUND: This parameter points to a sound file which is played whenever a word is successfully circled. Be sure to read the FAQ section of sound files. The default is that no sound is used.

  • LISTNAME: This parameter is simply a title or category which the hidden words all fall under. The default is just the term, "WORDSEARCH".

  • PUZZLE.DRAWNLOOPCOLOR: This parameter indicates the color of the loops which circle words after they have been drawn. See guidelines for colors to learn acceptable values for this parameter. The default color is black.

  • PUZZLE.BGCOLOR: This parameter indicates the background color of the applet's actual board where the hidden words appear. See guidelines for colors to learn acceptable values for this parameter. The default color is to use the value of the BGCOLOR parameter.

  • PUZZLE.COLUMNS: This parameter indicates the number of columns of letters used in the main board. The default is 10.

  • PUZZLE.FGCOLOR: This parameter indicates the foreground color of the applet's actual board where the hidden words appear. See guidelines for colors to learn acceptable values for this parameter. The default color is to use the value of the FGCOLOR parameter.

  • PUZZLE.LOOPCOLOR: This parameter indicates the color of the loops which circle words as they are being drawn. See guidelines for colors to learn acceptable values for this parameter. The default color is yellow.

  • PUZZLE.ROWS: This parameter indicates the number of rows of letters used in the main board. The default is to use the value of the PUZZLE.COLUMNS parameter.

  • SOLVED.SOUND: This parameter points to a sound file which is played whenever all of the words have been found (the puzzle is solved). Be sure to read the FAQ section of sound files. The default is that no sound is used.

  • WORDLIST: This parameter lists words which are to be hidden in the puzzle. Entries should be separated by whitespace (thus individual entries should not have spaces). The applet will try to hide as many of the words as possible starting with the first and moving down the list. However it will stop placing words if it finds the constraints (ROWS/COLUMNS) of the too small.
