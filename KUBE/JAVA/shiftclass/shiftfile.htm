Here is detail description of "shiftfile.class" Applet usage (you should know basics about how to include applets in your html files).

The purpose of this Applet is to change your output strings one by one, creating ullusion of movement on screen (see examples). Iit takes your output strings from file, gives you a possibility to specify different URL and direction of shift for every row in your data file.

Note: this applet is using Delimiter.class, to parse your strings, it means this file should be accessible from shiftfile.class (in other words, put both of them in the same directory).

<applet code=shiftfile.class width=600 height=30 >
  . shiftfile.class should be in lower case (at least for Windows NT and 95). width and height are very important mandatory parameters: they define the size of the Applet (in pixels). Do not forget to appropriately change codebase if you put shiftfile.class file in different directory than your source html file directory.
Tip: if you want to put some image as background for your Applet, usually you'll set wigth and height equal to the dimension of your image, otherwise you'll be able to see only part of your image (of course you can do it intentionally).
<param name=filename value="filename.txt">
  Put here the location and name for your Data File (apply usual rules: ".." for one directory up, "/" as separator and so forth). Data file shoud be in ASCII format (it means usual text file, what you create with Notepad) and should be on the same host as your source file and the Applet and should have appropriate access permissions, otherwise any applet will not be able to load this file. Every line of this ASCII file may have three section separated by sign "|". First part is output string itself, second is URL to go to if user clicks inside of your output area (see below) and third part is direction of shift two neighbour's strings: left for shift from RIGHT to LEFT(default), right - from LEFT to RIGHT, up - from DOWN to UP, down - from UP to DOWN

Your output string number 1 |http://www.anything.com/index.html|up
Your output string number 2 |ftp://ftp.something.com/|down
Your output string number 3 |gorher://whatever.address/|right
Your output string number 4 ||left

If you do not want to include any URL, you can put two "|" symbols in a row, like in example's string number 4. You don't have to include second (URL) and third (direction) sections at all, if you want.

<param name=x value=0>
<param name=y value=0>
<param name=outputwidth value=200>
<param name=outputheight value=200>
  Those four parameters define the location an size (in pixels) of output area inside of the Applet. They are self-explanatory: x and y are left-upper corner , outputwidth and outputheight are respectiveley width and height of the output area.You output strings will be centered inside of this area. You can apply your common sense to set them up right: x+outputwidth and y+outputheight should not be more than respectively width and height of the Applet (if they are, outputwidth will be set up as width - x and outputheight will be equal height - y). If you do not include some of them (or all of them) Applet will use default values: x=0, y=0, outputwidth=width, outputheight=height. Applet will change to those default values any wrong value it has gotten from you. It means that if you see your output string running at different area than you wanted, check out your coordinates: maybe your x is more than width or y is more than height, etc. Such "double" coordinate system gives you full control over location and size of the Applet itself (width and height) and location and size of your output strings and lets you create, for instance, your own background images with special place for output strings (see examples). Another thing: if you included outputheight parameter which is not equal 0 and its value is less than height of the font (either defaut or chosen by you), outputheight will be increased up to Font height. The result is very useful Tip: if you want outputheight to be equal to the Font height, set it up as 1 (if you set it up as 0 or do not include it at all, it will be set up equal to the Applet's height).
<param name=sleeptime value=300>
  sleeptime defines time in miliseconds between changing your output strings. Default value is 2000 (it means 2 seconds).
<param name=fontname value=Courier>
<param name=fontsize value=15>
<param name=bold value=1>
<param name=italic value=1>
<param name=forered value=255>
<param name=foregreen value=0>
<param name=foreblue value=255>
  Those settings will let you choose type, size and color of your font. Do not use them if system font is good enough for you. fontname is a logical name of font. Logical means that there are some predefined names, which are close, but not the same, for different platform. Usual values are: Dialog, Courier, Helvetica, DialogInput, TimesRoman. You can use fontlist applet to check out logical font names you have on your system. fontsize is size of the font in points (usual values between 10 and 12). Set up bold and italic parameters as 1 if you want your font in bold or italic. forered, foregreen and foreblue let you specify the RGB value of color for your font. You have to include all three parameters with valid values (from 0 to 255) if you want to change default color of your output string. If you set up even one value wrong, for example like foregreen=256, color of font will not be changed. Before trying to use your font, the Applet will get all fonts for every particular system and if it does not find matching entry, it uses system default font.
<param name=backred value=0>
<param name=backgreen value=0>
<param name=backblue value=255>
  Those parameters will let you set up the RGB value of color for the Applet background. Usually you'll omit these values if you have background image (but not necessarily - if image dimension is less that the Applet's one, you'd better set up some background to make it look nice and the same for diffeerent browser, because default background color differs from browser to browser (usually lightgray or white) - see example of using background color as well as image). Again, you have to include all three parameters with valid values (from 0 to 255) if you want to change background color of the Applet. If you set up even one value wrong, background color will not be changed.
<param name=image value="back.gif">
  image parameter lets you specify gif or jpg file as background for the Applet. The file should be on the same host as your html source file and should have appropriate access permissions, otherwise any applet will not be able to load this file. You can use .. symbols (two dots) as one directory up, like: value="../../somefile.gif" to look for file somefile.gif two directory up from current location of your source html file.
<param name=speed value=2>
  speed could be anywhere between 1 and 10 and this parameter defines how fast two strings will be shifted when the Applet will change your output strings. Defaul value is 5, 1 is slowest speed, 10 is fastest. Usually you do not have to set up this parameter, unless you want to experiment.
